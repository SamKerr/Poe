# Epic: Frontend UI

## Goal

Deliver a basic functioning web UI for the poetry service using server-rendered pages.

## Why this exists

The API is usable, but users need a simple first-party interface to read daily poems, browse history, and publish new poems.

## Scope

- Landing page showing today's poems only
- History page listing only days that have poems
- Day detail page for a selected history day
- Write page with poem submission flow
- Minimal in-app CSS styling

## Out of scope

- SPA frontend framework
- User accounts/auth
- Likes, comments, or social graph features

## Done when

A user can open the app in a browser, read today's poems, browse poem days, open a day view, and submit a poem via UI.
