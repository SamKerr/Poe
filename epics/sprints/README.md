# Sprint Planning

Use this folder for active sprint plans (for example `sprint-2.md`, `sprint-3.md`).

Conventions:

- One markdown file per sprint
- File name format: `sprint-<n>.md`
- Include goal, scope, agent assignment, sequencing, and done criteria
- Use `sprint-template.md` as the starting point
- Move completed sprint files into `epics/sprints/done/`

Required process:

1. Create/update active sprint file in `epics/sprints/`
2. Implement and verify sprint scope
3. Move completed sprint file to `epics/sprints/done/`
4. Move completed epic folder to `epics/done/` when epic scope is fully done
